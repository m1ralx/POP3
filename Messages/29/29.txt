*Замечания*:

1) строка 4
var c1 = 0; c2 = 0, c3 = 0, c4 = 0;
С такой записью var повлияет лишь на переменную с1
+ плохие имена переменных
+ лучше использовать запись вида:

*var len = 5,*

*      emptyStr = "",*

*      anotherStr = "Another string"; *
2) В строке 14 артефакт в виде буквы s
3) для строк 16-34 не помешал бы комментарий, поясняющий смысл кода
4) строка 49  не помещается на экран, к тому же ее можно (нужно) переписать
гораздо короче; обратить внимание и на строки 158, 164
5) switch на 53 строке: много дубляжа кода: нужно избавиться от этого
6) строки 132-135 выглядят печально. Иного способа склеить элементы массива
в строку нет?
7) у методов ToPostfix и ToInfix много общего. Создание массива с
приоритетами, проверку на некорректный ввод можно было бы вынести в общий
метод?
8) методы в javascript принято называть с маленькой буквы
9) var fso = new ActiveXObject('Scripting.FileSystemObject'); - нет смысла
дублировать эту строку
10) файл для вывода необязательно должен существовать при запуске. Что
мешает его создать в самой программе?
*11) *(A+B)*(C-D)^*(-(G+K))**L+c^d^e
Эту строку при попытке перевести в постфикс, а потом обратно в инфикс
превращает в
(A+B)*(C-D)^*((-G+K))**L+c^d^e

открывающиеся+закрывающиеся скобки выглядят плохо: *((-G+K))*
